📖 La importancia de las etiquetas semánticas

HTML5 introduce elementos como <header>, <nav>, <article>, <section>, <aside> y <footer> para describir el significado del contenido, mejorando la accesibilidad, SEO y mantenibilidad del código.

📌 ¿Qué es una sección?

El elemento <section> representa una agrupación genérica de contenido temáticamente relacionado. En este caso, cada sección dentro del artículo refuerza la estructura lógica. Ideal para capítulos o partes diferenciadas.

<section>
  <h2>Beneficios</h2>
  <p>Mejor accesibilidad...</p>
</section>

🎯 Ventajas para motores de búsqueda

Google y otros buscadores interpretan mejor la jerarquía de contenidos cuando se usan etiquetas semánticas. article y section ayudan a estructurar el documento de forma lógica, mejorando el posicionamiento.

Además, lectores de pantalla permiten a usuarios con discapacidad visual navegar por los roles de landmark (banner, navigation, main, complementary, contentinfo).

🧩 Diferencias entre article y section

<article> es para contenido independiente y distribuible (ej: post de blog, noticia). <section> agrupa contenido relacionado dentro de un artículo o en cualquier parte. Ambos pueden anidarse.

✅ En esta página, <article> contiene múltiples <section>, demostrando la práctica recomendada.

⚡ Nota visual: Cada bloque semántico muestra una etiqueta flotante con su nombre (header, nav, article, section, aside, footer).

📚 Referencia semántica

  • <header> → Introducción o navegación.
  • <nav> → Enlaces de navegación principal.
  • <article> → Contenido independiente.
  • <section> → Bloques temáticos genéricos.
  • <aside> → Contenido lateral relacionado.
  • <footer> → Metadatos, información de cierre.

🔍 Estructura del documento

Basado en la imagen de referencia: contenedor global → encabezado, contenido (article + aside), pie. Implementado fielmente con semántica HTML5.

<body>
  <header>...</header>
  <nav>...</nav>
  <main>
    <article>+<section></section></article>
    <aside>...</aside>
  </main>
  <footer>...</footer>
</body>
